   485→## Architecture
   486→
   487→### LOIS Alignment
   488→
   489→The codebase is structured to mirror the LOIS (Limit Order Intent System) architecture for easy porting to Rust:
   490→
   491→| TypeScript | Rust LOIS | Purpose |
   492→|------------|-----------|---------|
   493→| `RuleIntent` | `Intent` | Stored rule + token config |
   494→| `TriggerEvaluationResult` | `TriggerEvaluationResult` | Dispatcher output |
   495→| `RuleEvaluationRequest` | NATS message | Dispatcher → Evaluator |
   496→| `RuleDispatcher` | Dispatcher binary | Trigger evaluation |
   497→| `TradeTrigger` | Evaluator | Trade execution |
   498→
