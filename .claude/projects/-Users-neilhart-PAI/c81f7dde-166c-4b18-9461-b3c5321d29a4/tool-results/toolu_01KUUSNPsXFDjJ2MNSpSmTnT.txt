   338→## Registry Maintenance
   339→
   340→### Adding a System
   341→1. Run `Discover` workflow on the repo
   342→2. Entry is automatically added here
   343→3. Set `Production: Yes/No` based on whether it's in user-facing critical path
   344→4. If `Production: No`, add `Supports:` field listing production systems it affects
   345→5. Cross-system maps updated if connections found
   346→
   347→### Quality Ratings
   348→- **Excellent**: Complete ai.md, all sections filled, cross-system context documented
   349→- **Good**: ai.md exists with most sections, minor gaps
   350→- **Needs Update**: ai.md exists but stale or missing sections
   351→- **Not Mapped**: No ai.md, needs Discover workflow
   352→
   353→### Refresh Schedule
   354→- Re-run Discover after significant codebase changes
   355→- Review registry monthly for staleness
   356→- Update cross-system maps when integrations change
