# Top 10 AI Debugging Prompts: Summary

## Overview
This article provides 10 specialized prompts designed to help AI assistants deliver more effective debugging support across different scenarios and technologies.

## The 10 Prompts

### 1. **O(1) Chain-of-Thought Performance Analyzer**
**Purpose**: Optimize code performance to O(1) complexity

**Structure**:
- Phase 1: Component Identification
- Phase 2: Complexity Analysis (Big O notation breakdown)
- Phase 3: Optimization Opportunities
- Phase 4: System-Wide Impact Analysis

**Key Output**: Performance bottlenecks with proof of O(1) achievement and implementation code

---

### 2. **Claude-Style Logical Debugger**
**Purpose**: Find subtle bugs and edge cases through systematic reasoning

**Structure**:
- Layer 1: Logical Flow Analysis (preconditions, steps, postconditions)
- Layer 2: State Management (operation tracking)
- Layer 3: Edge Case Analysis
- Layer 4: Error Propagation tracing

**Key Output**: Categorized logical issues with severity levels and test verification

---

### 3. **Gradio Application Analyzer**
**Purpose**: Debug UI/backend integration in Gradio apps

**Structure**:
- Layer 1: Interface Component Analysis
- Layer 2: Event Flow Tracing
- Layer 3: State Management & Race Conditions

**Key Output**: Component issues, event chain vulnerabilities, and fixes

---

### 4. **Streamlit Application Debugger**
**Purpose**: Optimize Streamlit app flow and state management

**Structure**:
- Phase 1: Session State Analysis
- Phase 2: Component Hierarchy
- Phase 3: Performance Optimization (@st.cache strategies)

---

### 5. **Python-Specific Analyzer**
**Purpose**: Identify Python language-specific patterns and performance issues

**Structure**:
- Layer 1: Pythonic Pattern Analysis
- Layer 2: Memory Management
- Layer 3: Standard Library Usage optimization

---

### 6. **Frontend Debug Assistant**
**Purpose**: Fix UI/UX issues and browser compatibility problems

**Structure**:
- Layer 1: DOM Structure Analysis
- Layer 2: Browser Compatibility checking

---

### 7. **Database Query Optimizer**
**Purpose**: Optimize SQL queries and index strategies

**Structure**:
- Layer 1: Query Analysis (SQL structure, table access patterns)
- Layer 2: Index Strategy (existing indexes, new recommendations)

**Key Output**: Optimized SQL with expected performance improvements

---

### 8. **Security Vulnerability Analyzer**
**Purpose**: Identify and fix security vulnerabilities

**Structure**:
- Layer 1: Vulnerability Scanning
- Layer 2: Mitigation Strategy

**Categories**: Input validation, authentication, authorization, data protection

---

### 9. **API Integration Debugger**
**Purpose**: Handle request/response issues and error management

**Structure**:
- Layer 1: Request/Response Analysis
- Layer 2: Error Handling (network failures, timeouts, rate limiting)

---

### 10. **Memory Leak Detective**
**Purpose**: Identify and fix memory leaks

**Structure**:
- Layer 1: Memory Usage Analysis (allocation trends, reference counting)
- Layer 2: Leak Detection (symptoms, diagnosis, prevention)

---

## Best Practices for Using These Prompts

1. **Preparation**: Gather error messages, document expected behavior, prepare minimal reproductions
2. **Implementation**: Test incrementally, document changes, verify in isolation
3. **Validation**: Run comprehensive tests, check side effects, monitor impact

## Quick Reference: Scenario-to-Prompt Mapping

| Scenario | Prompt | Key Feature |
|----------|--------|-------------|
| Performance Issues | #1 O(1) Analyzer | Complexity analysis |
| Logic Bugs | #2 Claude-Style Debugger | Systematic reasoning |
| UI Problems | #6 Frontend Assistant | Browser compatibility |
| Data Issues | #7 Database Optimizer | Query optimization |
| Security | #8 Security Analyzer | Vulnerability assessment |

## Key Design Principles

Each prompt includes:
- **Structured analysis methodology**
- **Clear output formats**
- **Verification/testing steps**
- **Edge case consideration**
- **Best practices alignment**
- **Iterative refinement cycles** (typically 4-5 passes)

These prompts can be **combined** for complex issues requiring multiple expertise areas.
